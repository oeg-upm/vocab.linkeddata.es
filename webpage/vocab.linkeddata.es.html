Here you can find the list of vocabularies that the Ontology Engineering Group (OEG) is developing and publishing on the Web.

The P-Plan ontology

PROV extension for linking Plans and parts of plans to their respective executions.

See the LOV page for this vocab (p-plan).

The Workflow Motif Ontology

Ontology for describing Workflow Motifs. Workflow Motifs outline the kinds of data-intensive activities that are observed in workflows (data-operation motifs) and the different manners in which activities are implemented within workflows (workflow-oriented motifs).

See the LOV page for this vocab (wfm).

The Workflow Invocation Ontology

Wf-invoc is a simple profile of the P-plan ontology to describe how workflow steps are invoked within a workflow execution.

See the LOV page for this vocab (wf-invoc).

The Research Object Optimization Ontology

RO-Optimization is an extension of the RO model in order to be able to describe ooptimizations of workflows and their results.

The Workflow Fragment Description ontology

P-Plan extension to represent workflow fragments and their relationships to workflow templates

SMART Protocols Ontology: Protocol Module

Smart PROTOCOL is an ontology designed to describe the minimum information to report a laboratory protocol.

The Workflow Infrastructure Conservation Using Semantics ontology

The WICUS ontology network is a set of ontologies designed to represent the information about the computational infrastructure involved on the execution of computational scientific experiments modelled as scientific workflows.

The WICUS Hardware Specs ontology

The WICUS Hardware Specs ontology is describes the hardware characteristics of a computational infrastructure and the hardware requirements of a software component. This ontology is part of the WICUS Ontology network.

The WICUS Scientific Virtual Appliance ontology

The WICUS Scientific Virtual Appliance ontology describes the virtual computational resources used for executing scientific experiments, usally as Virtual Machines deployed on a IaaS provider or a virtualization solution. This ontology is part of the WICUS Ontology network.

The WICUS Software Stack ontology

The WICUS Software Stack ontology have been developed to describe the software elements of a computational resource. These descriptions can be used to describe both, the already deployed software components and the software requiremens of a workflow. This ontology is part of the WICUS Ontology network.

The WICUS Workflow Execution Requirements ontology

The WICUS Workflow Execution Requirements ontology relates a Scientific Workflow to the execution requirements and dependencies for executing it. This ontology is part of the WICUS Ontology network.

Linked Data Rights (LDR)

The Linked Data Rights ontology provides the vocabulary for creating rights expressions for Linked Data resources by extending the ODRL ontology. Linked Data assets (RDF triples, graphs, datasets, mappings...) can be object of protection by the intellectual property law, the database law or its access or publication be restricted by other legal reasons...

See the LOV page for this vocab (ldr).

The OPMW ontology

OPMW is an OPMV, PROV and P-plan extension to model the executions and definitions of scientific workflows.

See the LOV page for this vocab (opmw).

The Research Object Ontology

This ontology shows how AO and ORE ontologies can be used together to define a ResearchObject. This ontology is further customized by the wf4ever ontology.

The Wfprov Ontology

The wfprov ontology shows how to express minimal provenance information about the execution of a workflow described using the wfdesc ontology.

The Wfdesc ontology

This ontology ("wfdesc") describes an abstract workflow description structure, allowing the description of a scientific workflow as a direct acyclic graph, or a dataflow. The Aemet Ontology Network

Ontology network for the AEMET project. This model describes meteorological information.

El Viajero Ontology

OPMO profile for a News and Blogs scenario. Extension for modeling travels and routes.

RTVE Ontology ('Radio Televisión Española' - Spanish Radio and Television) provides concepts that explain the content of television and radio of the group. Its purpose is to describe information from RTVE, interconnection of its elements and content, as well as the enrichment of their website (http://www.rtve.es/) thanks to these interrelated data. Non-Ontological Resource Metadata Vocabulary

This vocabulary allows (1) describing the non-ontological resource available, (2) including in the ontology generated the provenance information by extending the Ontology Metadata Vocabulary

The Ccon Ontology

The Ccon represents the main concepts about the Cerrado plant communities structure and dynamics. This ontology will be part of the GeoLinked Data Ontology network to enrich the web of data with geospatial information of the Cerrado Biome, especially with regard the wood plant dynamics.

The Fire Ontology

The Fire Ontology was developed to represent the scientific knowledge about fire behave focused to describe the characteristics, causes and effects of fire on wood plants in the Brazilian Cerrado Domain.
